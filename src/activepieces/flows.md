# 🛠 Flows チュートリアルガイド

**～繰り返し作業を自動化して生産性を高めよう～**

## 🎯 はじめに

Flowsを使うことで、日常のタスクを自動化し、複数のアプリケーションを連携させたワークフローを簡単に構築できます。プログラミング不要で操作も直感的です。

## 🚀 フローの基本構造

フローは、以下の3ステップで構成されます：

- **トリガー**：フローを開始するイベント
- **アクション**：順番に実行される処理内容
- **応答やデータのやり取り**：外部サービスとの連携や結果の表示

## 🔔 トリガーの種類と使い方

フローを開始するイベントには複数のタイプがあります：

| トリガーの種類       | 説明                                                     |
| -------------------- | -------------------------------------------------------- |
| スケジュールトリガー | 毎日・毎時など、指定間隔で実行される                     |
| Webフックトリガー    | サイトやアプリでイベント（例：フォーム送信）を検知し起動 |
| 入力ステップ         | WebフォームやチャットUI経由でユーザーの入力を受けて開始  |
| 外部アプリ連携       | 例：新規メールを受信したタイミングで起動                 |

## 🔄 アクションの追加と設定

- テキスト生成（AI）、CRM統合（HubSpot）など多様なアクションが選択可能
- 接続パネルから外部サービスをリンク
- データセレクターで前のステップからの情報を次へ渡す（例：AI生成テキスト→メール本文へ）

## 🤖 AIとの連携方法

- **Ask AI ステップ**を利用して、ClaudeやChatGPTなどのLLMを組み込む
- APIキーを入力することで独自のAIサービスと連携可能

## 🧪 実行ログの確認

フローが実行された内容は「実行ログ」で詳細に確認可能です。ステップごとの処理結果をチェックして、問題の特定や改善が行えます。

## ✨ おわりに

Flowsを活用すれば、業務の自動化によって手間や時間を大幅に節約できます。最初はシンプルなトリガーとアクションから始め、徐々に複雑なフローを構築してみましょう。
